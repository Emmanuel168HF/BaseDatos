### 2. Descripción del Sistema de Base de Datos

El sistema administra:

- Clientes
- Ventas
- Productos
- Detalle de ventas

Permite registrar clientes, generar ventas y controlar los productos vendidos en cada transacción.

## 4. Diccionario de Datos

******Tabla:** Cliente

**Descripción:** _Almacena la información de los clientes._

| Campo | Tipo | Longitud | Restricciones | Descripción |
|:---|:---|:---:|:---|:---|
| IdCliente | INT | - | PK, AI, NN | Identificador único del cliente. |
| Nombre | VARCHAR | 100 | NN | Nombre del cliente. |

---

******Tabla:** Venta

**Descripción:** _Almacena la información de las ventas realizadas._

| Campo | Tipo | Longitud | Restricciones | Descripción |
|:---|:---|:---:|:---|:---|
| IdVenta | INT | - | PK, AI, NN | Identificador único de la venta. |
| Fecha | DATE | - | NN | Fecha de la venta. |
| Detalle | VARCHAR | 200 | Null | Observaciones de la venta. |
| IdCliente | INT | - | FK, NN | Cliente que realizó la compra. |

---

******Tabla:** Producto

**Descripción:** _Almacena la información de los productos._

| Campo | Tipo | Longitud | Restricciones | Descripción |
|:---|:---|:---:|:---|:---|
| IdProducto | INT | - | PK, AI, NN | Identificador único del producto. |
| Nombre | VARCHAR | 100 | NN | Nombre del producto. |
| Precio | DECIMAL | - | NN, CK (>0) | Precio del producto. |

---

******Tabla:** Contiene

**Descripción:** _Relaciona las ventas con los productos vendidos._

| Campo | Tipo | Longitud | Restricciones | Descripción |
|:---|:---|:---:|:---|:---|
| IdVenta | INT | - | PK, FK, NN | Venta registrada. |
| IdProducto | INT | - | PK, FK, NN | Producto vendido. |
| Cantidad | INT | - | NN, CK (>0) | Cantidad de productos vendidos. |
| PrecioProducto | DECIMAL | - | NN | Precio del producto al momento de la venta. |

---

### 5. Relaciones

| Columna 1 | Columna 2 | Columna 3 |
|:----------|:--------:|--------------------------------------:|
| Cliente -> | Venta | 1:N \| Un cliente puede realizar varias ventas. |
| Venta -> | Contiene | 1:N \| Una venta puede contener varios productos. |
| Producto -> | Contiene | 1:N \| Un producto puede formar parte de varias ventas. |

---

### 6. Matriz de Claves Foráneas

| Tabla | Campo FK | Referencia |
|:----------|:--------:|--------------------------------------:|
| Venta | IdCliente | Cliente(IdCliente) |
| Contiene | IdVenta | Venta(IdVenta) |
| Contiene | IdProducto | Producto(IdProducto) |

---

### 7. Integridad Referencial

| Regla | Descripción |
|:----------|:--------------------------------------|
| IR-01 | No se puede registrar una venta para un cliente inexistente. |
| IR-02 | No se puede agregar un producto inexistente a una venta. |
| IR-03 | No se puede eliminar una venta o un producto mientras existan registros en la tabla Contiene. |

---

### 8. Reglas del Negocio

- Cada cliente puede realizar múltiples ventas.
- Cada venta debe pertenecer a un solo cliente.
- Una venta debe contener al menos un producto.
- Un producto puede aparecer en varias ventas.
- La cantidad de productos vendidos debe ser mayor que cero.
