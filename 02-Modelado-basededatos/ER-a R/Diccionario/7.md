### 2. Descripción del Sistema de Base de Datos

El sistema administra:

- Alumnos
- Credenciales
- Materias
- Cursos
- Departamentos
- Profesores
- Proyectos
- Participaciones
- Dependientes

Permite controlar la información académica de los alumnos, los profesores, los departamentos y la participación de los docentes en proyectos.

### 5. Relaciones

| Columna 1 | Columna 2 | Columna 3 |
|:----------|:--------:|--------------------------------------:|
| Alumno -> | Credencial | 1:1 \| Un alumno tiene una credencial. |
| Alumno -> | Curso | 1:N \| Un alumno puede cursar varias materias. |
| Materia -> | Curso | 1:N \| Una materia puede ser cursada por varios alumnos. |
| Departamento -> | Profesor | 1:N \| Un departamento tiene varios profesores. |
| Departamento -> | Proyecto | 1:N \| Un departamento administra varios proyectos. |
| Profesor -> | Participa | 1:N \| Un profesor participa en varios proyectos. |
| Proyecto -> | Participa | 1:N \| Un proyecto tiene varios profesores. |
| Profesor -> | Dependiente | 1:N \| Un profesor puede tener varios dependientes. |

### 7. Integridad Referencial

| Regla | Descripción |
|:----------|:--------------------------------------|
| IR-01 | No se puede generar una credencial para un alumno inexistente. |
| IR-02 | No se puede registrar un curso para un alumno o materia inexistentes. |
| IR-03 | No se puede registrar un profesor en un departamento inexistente. |
| IR-04 | No se puede crear un proyecto para un departamento inexistente. |
| IR-05 | No se puede asignar un profesor a un proyecto inexistente. |
| IR-06 | No se puede registrar un dependiente para un profesor inexistente. |

---

### 8. Reglas del Negocio

- Cada alumno posee una única credencial.
- Un alumno puede cursar varias materias.
- Cada materia es impartida por un profesor.
- Todo profesor pertenece a un departamento.
- Cada departamento administra varios proyectos.
- Un profesor puede participar en varios proyectos.
- Un profesor puede registrar varios dependientes.
- Ningún registro puede hacer referencia a una clave inexistente.
